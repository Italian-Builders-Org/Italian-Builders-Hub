/goal
Build the first static landing page for “Italian Builders”, a young Italian community of people building apps, startups, AI tools, SaaS products, open-source projects, and internet businesses.

The goal of this first version is NOT to build the full community platform yet.
The goal is to create a polished public-facing website that:
1. Explains what Italian Builders is.
2. Makes the community feel active, ambitious, and credible.
3. Collects emails for the upcoming community platform.
4. Shows that builder profiles, project pages, categories, and community OS projects are coming soon.
5. Uses a visual direction inspired by Contra’s project/community browsing experience, but without copying it directly.

Reference:
- Contra projects/community style: https://contra.com/?view=projects
- Existing rough community page: https://italianbuilders.lovable.app/

Core positioning:
Italian Builders is the home for Italian builders creating apps, startups, AI products, SaaS tools, open-source projects, and internet businesses.

Tone:
Modern, sharp, energetic, community-first, slightly editorial.
Avoid corporate startup clichés.
Avoid sounding like an incubator, accelerator, or LinkedIn clone.
The site should feel like a movement of people who build.

Visual direction:
- Clean, modern, high-density but readable.
- Inspired by Contra: cards, project grids, category filters, featured sections, people/projects as the main content.
- Use strong typography.
- Use a light theme by default.
- Add subtle gradients, soft borders, rounded cards, and tasteful motion.
- The homepage should feel like a live directory/product showcase even if the real data is still mocked.
- Mobile-first responsive design.

Tech requirements:
- Build as a static landing page.
- Use the existing app stack if present.
- If this is a React/Vite/Next project, implement using the existing structure.
- Use reusable components.
- No backend required for now.
- The waitlist form can submit to a placeholder function or use a mocked handler.
- Make it easy to later connect the form to Supabase, MailerLite, Resend, ConvertKit, or another provider.
- Do not over-engineer authentication, profiles, or project submission in this iteration.

Copy constraints:
- Keep text concise.
- Avoid long paragraphs.
- Prefer punchy section intros.
- Use English as the main language for the first version, because the community may include Italian builders who want international visibility.
- It is okay to use “Italian Builders” as a proper noun.
- Do not use fake metrics beyond “240+ builders joined in the first 48 hours.”
- Make placeholder content visually useful but clearly replaceable.

Implementation details:
- Create a clean component structure:
  - Header
  - Hero
  - CategoryPills
  - ProjectCard

  - features
  - roles
This will make it easy to replace mock data later.

Important:
Do not build login.
Do not build real user profiles.
Do not build a dashboard.
Do not build a CMS.
Do not add payment or membership yet.
This is only the first landing page to validate interest and collect emails.

Final result:
A polished, responsive, production-ready landing page for Italian Builders that feels like the beginning of a serious builder network, with a strong Contra-inspired project/community browsing feel.
